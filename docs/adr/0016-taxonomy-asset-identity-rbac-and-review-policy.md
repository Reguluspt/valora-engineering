# ADR 0016 - Taxonomy and Asset Identity RBAC and Review Policy

## Status
Proposed

## Context
Suggestions generated by similarity matchers represent system proposals and must not update official master data directly. Human decision-making remains the source of truth for taxonomy updates and asset approvals.

## Decision
1. **RBAC Scopes Matrix**:
   - `taxonomy:node:create` / `taxonomy:node:update` / `taxonomy:node:approve` / `taxonomy:node:deprecate`
   - `taxonomy:variant:create` / `taxonomy:variant:update` / `taxonomy:variant:approve`
   - `asset_identity:suggest` / `asset_identity:batch_approve` / `asset_identity:merge` / `asset_identity:alias:create`
2. **Review Integrity**:
   - Candidates created by similarity scoring must maintain a `review_status = pending`.
   - Batch approval endpoints validate confidence parameters and reject candidates carrying conflict warnings.
   - Audit trail entries are logged in the `IdentityDecisionLog` upon any approval execution.

## Consequences
- Prevents database pollution from unverified AI or rule-based suggestions.
- Auditable accountability for master data changes.
